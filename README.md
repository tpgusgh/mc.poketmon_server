# mc.poketmon_server

포켓몬(Pixelmon, 마인크래프트 1.16.5 + Forge) 서버를 위한 웹 대시보드 + 백엔드 API + 자체 제작 게임 모드의 소스코드입니다.

## 구성

- **`backend/`** — FastAPI 백엔드
- **`frontend/`** — React + Vite 웹 대시보드
- **`factionbuff-mod/`** — 진영 경험치 버프를 실제 게임 안에 적용하는 소형 Forge 모드 (자세한 설명/빌드 방법은 해당 폴더의 README 참고)

## 실제 구현된 기능들

### 접속 현황 / 대시보드
- 플레이어별 실시간 온라인·오프라인 상태 카드
- 접속 시간 표시 (일/시간/분 단위 자동 변환, 예: "2일 19시간 0분")
- 통합 대시보드(드롭다운으로 종류 전환): 누적 접속시간 순위, 바보같은 죽음(낙사 포함), 전설 포켓몬 보유 수, 업적 달성 개수
- 서버 CPU/메모리 실시간 상태 표시

### 서버 스토리
- 접속/퇴장, 업적 달성, 낙사, 포켓몬 포획/교환, 배틀 결과 등 서버 로그를 날짜별로 묶어서 "이날 무슨 일이 있었다"는 문장으로 자동 생성 (LLM 호출 없이 순수 규칙 기반)
- 그날 있었던 이벤트 내용에 따라 다양한 하루 요약 헤드라인이 붙음 (날짜별로 고정된 랜덤이라 같은 날은 항상 같은 문구)
- 스크롤 무한 로딩 (한 번에 5일씩 불러오고, 스크롤을 내리면 더 불러옴)

### 전설 포켓몬 추적
- 포켓몬 개체 고유 UUID를 스냅샷 비교해서 포획/교환을 실시간으로 정확히 감지 (업적에만 의존하지 않음)
- 파티(들고 다니는 포켓몬) + PC박스까지 전부 스캔
- 전체 국가도감 1025종 한글 정식 명칭 사용

### 진영 시스템 (발로 / 미스틱 / 인스팅트 / 하모니)
- 회원가입할 때 진영을 하나 골라야 하고, 한 번 정하면 변경 불가
- 진영별로 소속 플레이어들의 배틀 랭킹 점수 **평균**을 매겨 순위 산정
- **1등 진영은 실제 게임 안에서 배틀 경험치 1.5배, 2등 진영은 1.2배** — 아이템 지급이 아니라 자체 제작 Forge 모드(`factionbuff-mod`)가 픽셀몬의 경험치 이벤트를 직접 후킹해서 정확한 배수를 적용
- 순위가 바뀌면 서버 재시작 없이 자동으로 반영
- "플레이어별 파티 포켓몬" 섹션에서 각 플레이어 닉네임이 소속 진영 색으로 표시되고 진영 이름 배지가 붙음
- **마인크래프트 채팅에도 진영 색이 적용됨** — 채팅을 치면 "[발로] kimhyunhoking: 메시지"처럼 진영 이름 부분만 그 진영 색(빨강/파랑/노랑/연보라)으로, 닉네임과 본문은 흰색으로 표시 (`factionbuff-mod`가 `ServerChatEvent`를 후킹해서 실시간으로 재구성)

### 배틀 신청/랭킹 시스템
- 로그인 + **본인도 상대도 지금 게임에 접속 중이어야** 배틀 신청 가능
- 신청은 1시간 동안만 유효, 시간 지나면 자동 만료 (다시 신청해야 함)
- 보낸 신청 취소 가능
- 신청/수락 시 마인크래프트 채팅에 전체 알림이 뜨고, 수락 시 배틀 당사자 둘을 정해진 배틀장 좌표로 자동 텔레포트(RCON)
- 관전하고 싶은 사람은 채팅의 "[관전하러가기]"를 클릭하면 상대에게 텔레포트 요청(`/tpa`)이 가고, 상대가 수락하면 이동 (일반 플레이어 권한으로도 문제없이 동작하도록 FTB Essentials의 텔레포트 요청 기능을 재사용)
- 같은 상대와는 하루 최대 3판까지만 가능 (친구끼리 랭킹 파밍 방지)
- 승패는 양쪽이 각자 자기 결과를 보고 → 서로 일치하면 자동 반영, 불일치하거나 30분간 무응답이면 관리자가 최종 판정
- 배틀 랭킹: 시작 1000점, 승리 시 +14~17점, 패배 시 -12~-15점. 아직 배틀 안 한 플레이어도 1000점으로 순위에 표시됨. 로그인한 상태로 보면 본인 이름이 초록색으로 강조 표시
- 헤더의 🔔 알림 벨로 받은 신청/배틀 결과를 바로 확인 가능 (빨간 배지로 안 읽은 개수 표시)

### 계정 시스템
- 아이디/비밀번호로 회원가입하면서 본인이 어떤 마인크래프트 플레이어인지 선택 (한 번 선택된 플레이어는 다른 계정이 재선택 불가)
- 이미 다른 사람이 선점한 닉네임을 발견하면, 로그인 없이도 "이의제기" 접수 가능 → 관리자가 심사해서 잘못된 선점이면 선택을 취소시킬 수 있음
- 비밀번호는 PBKDF2-SHA256(20만 회 반복) + 계정별 salt로 해시 저장
- 로그인 브루트포스 방지: 같은 아이디로 15분 내 5회 실패하면 15분간 잠금
- 세션 쿠키는 HttpOnly + Secure + SameSite=Strict

### 문의하기
- 로그인한 플레이어가 관리자에게 문의를 남기면, 관리자 전용 화면에서 누가 어떤 내용을 보냈는지 확인하고 처리 완료 표시 가능

### 관리자 전용 기능 (전부 서버 쪽에서 세션 기반으로 권한 검증)
- 배틀 결과 분쟁(자기보고 불일치/무응답) 최종 판정
- 계정 이의제기 심사(선택 취소 / 기각)
- 웹사이트에서 문구를 입력하면 **지금 접속 중인 모든 플레이어의 채팅창에 즉시 공지** 전송 (RCON)
- 문의함 확인 및 처리
- **플레이어 아이템(인벤토리) 백업/복구**: 서버가 재시작될 때마다 전체 플레이어 인벤토리를 자동 스냅샷(최근 7일치 보관). 공허 낙사 등으로 아이템을 잃어버렸을 때, 해당 플레이어가 오프라인일 때 특정 시점 스냅샷으로 복구 가능

### 접속 가이드
- 헤더의 "🎮 접속 방법" 버튼을 누르면 모달로 필요한 마인크래프트/Forge 버전, 클라이언트 설치 모드 목록, 서버 주소, 자동 재시작 시각, 서버 규칙(집 털기 금지 등)을 안내

## 폴더별 상세

- `backend/requirements.txt` — 백엔드 의존성 목록
- `backend/.env.example` — 백엔드가 필요로 하는 환경변수 (RCON 비밀번호, 쿠키 도메인) 예시
- `factionbuff-mod/README.md` — 왜 이 모드를 만들었는지, Gradle 없이 서버에 이미 있는 클래스들로 직접 `javac` 컴파일한 방법

## 참고

이건 특정 실제 운영 중인 개인 서버용 소스코드라, 범용 템플릿은 아닙니다 — 도메인, 파일 경로, 관리자 UUID 등이 코드 곳곳에 하드코딩되어 있습니다. 계정/세션/RCON 비밀번호 같은 런타임 데이터와 비밀 값은 의도적으로 버전관리에서 제외했습니다.
